KMeans Clustering Analysis

Introduction

This section applies KMeans clustering to identify hidden patterns among job postings. Since KMeans is an unsupervised learning method, it does not use a target variable. Instead, job postings are grouped based on salary and experience requirements.

NAICS industry classification is used as a reference label to compare whether the clusters align with existing industry categories.

Feature Selection

The KMeans model uses salary and experience variables because these features help describe job market segments from a job seeker perspective.

Code

Standardization

KMeans is distance-based, so the selected numerical features must be standardized before clustering.

Code 
scaler = StandardScaler()
scaled_data = scaler.fit_transform(cluster_df)

KMeans Clustering

Code 
kmeans = KMeans(
    n_clusters=4,
    random_state=42,
    n_init=10
)

cluster_df["Cluster"] = kmeans.fit_predict(scaled_data)

Cluster Distribution

Code 
cluster_counts = cluster_df["Cluster"].value_counts().reset_index()
cluster_counts.columns = ["Cluster", "Count"]
cluster_counts = cluster_counts.sort_values("Cluster")

cluster_counts["Cluster_Label"] = cluster_counts["Cluster"].map({
    0: "Cluster 0: Entry-Level Jobs",
    1: "Cluster 1: Mid-Level Jobs",
    2: "Cluster 2: Experienced Jobs",
    3: "Cluster 3: High Salary Jobs"
})

fig = px.bar(
    cluster_counts,
    x="Cluster",
    y="Count",
    color="Cluster_Label",
    title="Distribution of Job Postings by KMeans Cluster",
    text="Count"
)

fig.update_layout(
    xaxis_title="KMeans Cluster",
    yaxis_title="Number of Job Postings",
    height=500,
    xaxis=dict(
        tickmode="array",
        tickvals=[0, 1, 2, 3],
        ticktext=["0", "1", "2", "3"]
    ),
    legend_title_text="Cluster Meaning"
)

fig.show()

The clustering results indicate that most job postings belong to Cluster 1 and Cluster 2, suggesting that the labor market is dominated by medium- to high-demand positions. Cluster 0 represents a relatively small segment of postings, while Cluster 3 captures a specialized group with different salary and experience characteristics.

PCA Visualization of Clusters

PCA is used to reduce the standardized features into two dimensions so the clusters can be visualized.

Code 
pca = PCA(n_components=2)
pca_components = pca.fit_transform(scaled_data)

cluster_df["PCA1"] = pca_components[:, 0]
cluster_df["PCA2"] = pca_components[:, 1]
cluster_df["Cluster_Label"] = cluster_df["Cluster"].map({
    0: "Entry-Level Jobs",
    1: "Mid-Level Jobs",
    2: "Experienced Jobs",
    3: "High Salary Jobs"
})

fig = px.scatter(
    cluster_df,
    x="PCA1",
    y="PCA2",
    color="Cluster_Label",
    title="KMeans Clusters Visualized with PCA",
    hover_data=["NAICS2_NAME"]
)

fig.add_vline(
    x=0,
    line_width=2,
    line_dash="dash",
    line_color="gray"
)

fig.add_hline(
    y=0,
    line_width=2,
    line_dash="dash",
    line_color="gray"
)

fig.update_layout(
    xaxis_title="PCA Component 1",
    yaxis_title="PCA Component 2",
    height=600,
    legend_title_text="Cluster"
)

fig.show()

The PCA plot shows that the four clusters are reasonably separated after dimensionality reduction, suggesting that KMeans identified meaningful group structures.

Interpretation

The KMeans model groups job postings based on salary and experience requirements. The NAICS industry labels are used as reference labels to help interpret whether clusters are associated with specific industry categories.

This analysis suggests that jobs with similar salary and experience profiles can be grouped into meaningful labor market segments. Comparing clusters with NAICS industries helps connect the unsupervised model results to real-world industry classifications.

These findings demonstrate that salary and experience variables alone can reveal meaningful labor market segments using unsupervised learning.

Implications for Job Seekers

For job seekers, clustering can help identify different types of career opportunities. Some clusters may represent lower-experience or entry-level roles, while others may represent higher-salary or more experienced positions.

Understanding these clusters can help students and professionals compare career paths, evaluate salary expectations, and identify which industries may offer stronger opportunities for data science, business analytics, and machine learning roles.
